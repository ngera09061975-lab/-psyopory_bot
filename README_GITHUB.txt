=== ИНСТРУКЦИЯ ДЛЯ НОВИЧКА: GITHUB + БОТ ===

ВАЖНО: GitHub — это облачная копия вашего кода.
Он запоминает все изменения. Если вы сломали что-то — можно вернуть старое за 1 клик.

═══════════════════════════════════════════════════════════
ШАГ 1. РЕГИСТРАЦИЯ НА GITHUB (5 минут)
═══════════════════════════════════════════════════════════

1. Откройте https://github.com
2. Нажмите зелёную кнопку "Sign up" (Регистрация)
3. Введите email, придумайте пароль, подтвердите email
4. На вопросы "What do you want to do?" — можно нажимать "Skip personalization"
5. Готово! Вы внутри GitHub.

═══════════════════════════════════════════════════════════
ШАГ 2. СОЗДАЁМ ХРАНИЛИЩЕ (РЕПОЗИТОРИЙ)
═══════════════════════════════════════════════════════════

1. На главной странице GitHub нажмите зелёную кнопку "New" (или "Create repository")
2. Название: nadezhda-bot
3. Описание: Бот для психолога — лид-магнит, прогрев, оплата
4. ВАЖНО: поставьте галочку "Private" (Частный) — код не будет виден посторонним
5. НЕ ставьте галочку "Add a README" — у нас уже есть файлы
6. Нажмите "Create repository"
7. Готово! Вы увидите пустую страницу с инструкцией.

═══════════════════════════════════════════════════════════
ШАГ 3. УСТАНАВЛИВАЕМ GITHUB DESKTOP (ПРОГРАММА ДЛЯ КОМПЬЮТЕРА)
═══════════════════════════════════════════════════════════

Это программа с кнопками — без командной строки, всё как в обычных программах.

1. Откройте https://desktop.github.com
2. Скачайте установщик для Windows (или Mac)
3. Установите — везде жмите "Далее"
4. Откройте GitHub Desktop
5. Войдите: нажмите "Sign in to GitHub.com" → введите логин/пароль от шага 1

═══════════════════════════════════════════════════════════
ШАГ 4. КОПИРУЕМ ФАЙЛЫ БОТА В ПРОГРАММУ
═══════════════════════════════════════════════════════════

1. В GitHub Desktop нажмите "File" → "Clone repository" (Скопировать хранилище)
2. Выберите вкладку "GitHub.com" — вы увидите свой "nadezhda-bot"
3. Нажмите на него
4. В поле "Local path" выберите папку на компьютере, куда сохранить
   (например: Документы/GitHub/nadezhda-bot)
5. Нажмите "Clone" (Скопировать)
6. Готово! У вас появилась папка на компьютере, связанная с GitHub.

Теперь:
- Возьмите файлы из этого архива (main.py, .env.example, requirements.txt, .gitignore)
- Скопируйте их в эту папку (Документы/GitHub/nadezhda-bot)
- Создайте внутри папку audio/ и положите туда свои mp3

═══════════════════════════════════════════════════════════
ШАГ 5. ЗАГРУЖАЕМ ФАЙЛЫ НА GITHUB (ПЕРВЫЙ РАЗ)
═══════════════════════════════════════════════════════════

1. Вернитесь в GitHub Desktop
2. Вы увидите список изменённых файлов слева
3. Внизу есть поля:
   - Summary (описание): напишите "Первая версия бота"
   - Description (подробности): можно оставить пустым
4. Нажмите кнопку "Commit to main" (Сохранить в main)
   Это значит: "запомни, что сейчас у меня такие файлы"
5. Нажмите кнопку "Push origin" (Отправить на сервер)
   Это значит: "отправь копию на GitHub в интернет"
6. Готово! Теперь ваш код в облаке. Если компьютер сломается — код не пропадёт.

═══════════════════════════════════════════════════════════
ШАГ 6. КАК МЕНЯТЬ ТЕКСТЫ ПРОГРЕВА (КАЖДЫЙ ДЕНЬ)
═══════════════════════════════════════════════════════════

1. Откройте папку на компьютере (Документы/GitHub/nadezhda-bot)
2. Откройте файл main.py в обычном Блокноте (или Notepad++ — лучше)
3. Нажмите Ctrl+F, введите: TOUCH_TEXTS
4. Найдёте блок с текстами касаний 1-6
5. Меняйте текст внутри кавычек. Например:
   Было:  "Ну как ты?"
   Стало:  "Привет! Как твои дела?"
6. Сохраните файл (Ctrl+S)
7. Откройте GitHub Desktop — он покажет, что файл изменился
8. В поле Summary напишите: "Обновила касание 3 — новый текст"
9. Нажмите "Commit to main" → "Push origin"
10. Готово! На GitHub теперь новая версия.

═══════════════════════════════════════════════════════════
ШАГ 7. КАК ДОБАВИТЬ НОВОЕ АУДИО
═══════════════════════════════════════════════════════════

1. Запишите аудио на телефоне
2. Переименуйте файл: touch_2.mp3 или touch_4.mp3 (или lead_magnet.mp3)
3. Положите файл в папку audio/ (внутри папки бота на компьютере)
4. Откройте GitHub Desktop
5. Напишите в Summary: "Добавила аудио для касания 2"
6. Commit → Push
7. Готово!

═══════════════════════════════════════════════════════════
ШАГ 8. КАК ВЕРНУТЬСТЯ НАЗАД (ЕСЛИ СЛОМАЛИ)
═══════════════════════════════════════════════════════════

1. В GitHub Desktop нажмите "History" (История) — сверху
2. Вы увидите все ваши изменения с датами
3. Найдите нужную старую версию, нажмите правой кнопкой
4. Выберите "Revert this commit" (Вернуть к этой версии)
5. Готово! Всё вернулось как было.

═══════════════════════════════════════════════════════════
ШАГ 9. ЗАГРУЖАЕМ БОТА НА ХОСТИНГ
═══════════════════════════════════════════════════════════

Вариант A — Bothost (проще):
1. Зайдите на bothost.ru, создайте проект Python
2. Нажмите "Upload files" (Загрузить файлы)
3. Выберите ВСЕ файлы из папки на компьютере (кроме папки data/)
4. В панели настроек (ENV) вставьте переменные из .env.example
5. Нажмите "Deploy" (Запустить)

Вариант B — Amvera:
1. Зарегистрируйтесь на amvera.ru
2. Создайте проект, скопируйте ссылку для Git
3. В GitHub Desktop: Repository → Repository settings → Remote
4. Добавьте ссылку Amvera как второй remote
5. Push — файлы улетят на хостинг автоматически

═══════════════════════════════════════════════════════════
ПРАВИЛА БЕЗОПАСНОСТИ
═══════════════════════════════════════════════════════════

- Файл .env (с токенами) НЕ загружается на GitHub — он в .gitignore
- На GitHub лежит только код, без паролей
- На хостинге вы вручную вводите токены в настройках (ENV)
- Если кто-то увидит ваш токен — он сможет управлять ботом. Храните в .env!

═══════════════════════════════════════════════════════════
ЧТО ДЕЛАТЬ, ЕСЛИ ЗАСТРЯЛИ
═══════════════════════════════════════════════════════════

- Не получается зайти в GitHub — проверьте email, возможно письмо в спаме
- GitHub Desktop не видит изменения — убедитесь, что файл сохранён (Ctrl+S)
- Не понятно, что такое Commit — это просто "снимок". Фотография ваших файлов.
- Не понятно, что такое Push — это "отправить фотографию в облако".
